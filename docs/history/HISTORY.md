## [v0.2.0] - 2026-02-07

### Added
*   新增 **PR 追踪功能**: 通过 `pr` 命令查询各练习的历史最高重量（PR）。
*   新增 **1RM 预估功能**: 在 PR 查询结果中，利用 **Epley** 与 **Brzycki** 公式自动计算并展示每组的预估 1RM 值。
*   新增 **训练周期查询**: 通过 `cycles` 命令列出数据库中的所有训练周期及其基本信息。
*   新增 **练习列表查询**: 通过 `list` 命令列出所有练习，支持 `--type` 参数进行分类筛选。
*   新增 **容量与运动学分析**: 通过 `volume` 命令查询指定周期内特定练习类型的深度统计，包括总容量、日均容量、**平均强度**、**训练频率**、**训练密度**及强度分布。
*   新增 **强化报告系统**: 自动生成 `Summary.md` 名人堂，汇报 PR 记录与 Dashboard。
*   支持 **多样化备注**: 支持日级（`r`）、活动及组级别备注（`//`, `#`, `;`），贯穿全链路。
*   新增 **粒度退出代码 (Granular Exit Codes)**: 引入 `AppExitCode` 枚举取代布尔返回，支持针对性错误状态识别。

### Changed/Refactor
*   重构 **构建输出架构**: 引入 `bin/` 目录方案，将二进制文件与中间产物分离。
*   标准化 **输出文件结构**: 统一 `output/data/` (JSON), `output/db/` (SQLite), `output/reports/` (Markdown)。
*   重构 **mytest 测试工具**: 模块化核心逻辑，通过 `run.py` 统一入口，支持 `test_fast.bat` 环境自适应。
*   重构 **参数处理**: 引入结构体传参，解决容易混淆的参数警告，并统一风格为 **PascalCase**。
*   重构 **错误传播链路**: 全面启用 `AppExitCode` 类型，使 `ActionHandler` 及各 Handler 具备精准的错误传导能力。
*   标准化 **CLI 退出逻辑**: `main_cli.cpp` 现在通过 `static_cast<int>` 吐出标准的操作系统退出码。
*   增强 **测试套件集成**: `StepExecutor` 内置退出码映射并在失败时展示高亮描述性信息。
*   技术栈性能优化: 在核心模型中明确 C++ 原生 `struct` 替代 JSON 通信的性能设计思路，减少转换开销。

### Fixed
*   修复了多个核心模块中的 `clang-tidy` 警告（幻数、隐式转换、命名规范）。
*   修复了构建脚本中路径处理不一致的问题。
